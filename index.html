SVG is an image format for vector graphics. It literally means Scalable Vector Graphics.

Why use SVG at all?

  • Small file sizes that compress well
  • Scales to any size without losing clarity (except very tiny)
  • Looks great on retina displays
  • Design control like interactivity and filters

Basic usage

Using SVG as an <img>

  <img src="gift.svg" width="128" height="128" alt="Gift">
  <img src="gift.svg" width="64" height="64" alt="Gift smaller">
  <img src="gift.svg" width="32" height="32" alt="Gift tiny">

Browser support

It works everywhere except IE 8 and down and Android 2.3 and down.

Using SVG as a background-image

  .svg-icon {
    display: block;
    width: 128px;
    height: 128px;
    background: url(store.svg);
    background-size: 128px 128px;
  }
  <span class="svg-icon"></span>

Browser support

It works everywhere except IE 8 and down and Android 2.3 and down.

Using "inline" SVG

  <div>
    <svg xmlns="http://www.w3.org/2000/svg" height="256" width="256" viewBox="0 0 100 100"><g>...</g></svg>
  </div>

Inline SVG is a part of DOM so you can control it with CSS! Hover over credit card below to see the result.

Browser support

Again, it works everywhere except IE 8 and down and Android 2.3 and down.

Using SVG as an <object>

  <object type="image/svg+xml" data="currency.svg" class="currency">
    <!-- Ideal place for fallback image in CSS -->
  </object>

Base64 encoded SVG in data URI

Encoded SVG can be used in the same manner as a normal one but with no additional HTTP request. This method is not recommended, see the next section.

As <img>

<img src="data:image/svg+xml;base64,[data]">

As CSS

.svg-icon {
  background: url("data:image/svg+xml;base64,[data]");
}

As <object>

<object type="image/svg+xml" data="data:image/svg+xml;base64,[data]">
  fallback
</object>

Raw SVG in data URI

Base64 encoded SVG can produce a larger result than the raw code, it is unreadable and not so flexible. Prefer the raw SVG in the next manner:

As <img>

<img src="data:image/svg+xml;utf8,<svg>[svg code]</svg>">

As CSS

.svg-icon {
  background: url("data:image/svg+xml;utf8,<svg>[svg code]</svg>");
}

As <object>

<object type="image/svg+xml" data="data:image/svg+xml;utf8,<svg>[svg code]</svg>">
  fallback
</object>

SVG "sprites"

SVG has a brilliant technique that is similar to bitmap sprites. It permit to use the collection of <symbol> elements in a single SVG instead making HTTP request for every image. You can define the viewBox directly on the <symbol> and then not need one when you <use> it later in an <svg>. Every <symbol> must have an ID to make the reference to the specific <symbol> in a collection with <use>.

SVG symbols collection in an external file

 <!-- Symbols collection definition. External icons.svg file -->
 <svg xmlns="http://www.w3.org/2000/svg">
   <symbol viewBox="0 0 100 100" id="facebook">
     <title>Facebook</title>
    <g>
      <circle ...
    </g>
  </symbol>
  <symbol viewBox="0 0 100 100" id="twitter">
    <title>Twitter</title>
    <g>
      <circle ...
    </g>
  </symbol>
</svg>
 <!-- Usage in HTML document -->
 <svg class="svg-sprite">
   <use xmlns:xlink="http://www.w3.org/1999/xlink" xlink:href="icons.svg#facebook"></use>
 </svg>
 <svg class="svg-sprite">
   <use xmlns:xlink="http://www.w3.org/1999/xlink" xlink:href="icons.svg#twitter"></use>
 </svg>

Inline symbols collection definition

<body>
   <!-- symbols definition -->
   <svg xmlns="http://www.w3.org/2000/svg" style="display: none;">
     <symbol id="facebook" viewBox="0 0 100 100">
       <title>original-file's-title or just friendly symbol's name</title>
       <description>Some short description to not forget what is this and where it from</description>
       <!-- symbol code -->
     </symbol>
     <symbol id="twitter" viewBox="0 0 100 100">
       <!-- symbol code -->
     </symbol>
   </svg>

   <!-- symbols usage -->
   <svg class="icon">
     <use xlink:href="#facebook" />
   </svg>
   <svg class="icon" style="width: 256px; height: 256px">
     <use xlink:href="#twitter" />
   </svg>
</body>

Browser support

There is the difference between external collections definition and inline. Inline definition has better support, it works in IE9+, Chrome 35+, FF 15+, Safari 7.1+, Android 40+, BB 10+. Collections in external files are not supported by IE at all. Luckily there is the polyfill svg4everybody that enables this functionality in all browsers that supports basic SVG and as a bonus provides fallback to .png in older browsers.

svg4everybody is only 64 sloc but there is even smaller solution that does almost the same:


   (/MSIE|Trident/.test(navigator.userAgent) || /Edge\/12/.test(navigator.userAgent) || (navigator.userAgent.match(/AppleWebKit\/(\d+)/) || [])[1] < 537) && document.addEventListener('DOMContentLoaded', function () {

        object.data = use.getAttribute('xlink:href');

      }

Fallbacks

There are a lot of solutions that provides fallback to SVG for older browsers like an IE8 or Android 2.x.

Javascript fallbacks

Javascript is the easiest way, just check for SVG support, add special CSS class and provide some CSS fallback.

// Vanilla JS checker
if (!document.createElementNS || !document.createElementNS('http://www.w3.org/2000/svg', 'svg').createSVGRect) {
    document.documentElement.className += ' no-svg';
}
// Check with Modernizr
if (!Modernizr.svg) {
  document.documentElement.className += ' no-svg';
}

Example usage

<head>
  .no-svg .icon {
    display: inline-block;
    width: 128px;
    height: 128px;
    background-image: url(icon.png);
  }
</head>
<body>
  <object type="image/svg+xml" data="icon.svg" class="icon"></object>
</body>

Alternative JS solution

<img src="medal.svg" onerror="this.src='medal.png';this.onerror=null;">

CSS only fallbacks

Fallback for SVG as <img>

There is no pure CSS solution for this. But there is the <image> trick.

<svg width="128" height="128">
  <image xlink:href="cart.svg" src="cart.png" width="128" height="128" />
</svg>

Fallback notes: Triggers multiple aborted requests in IE. Requires absolute values for width and height, relative values like a percentage can be interpreted as 0.

Fallback for SVG as <object>

This is the easiest way to provide fallback content.

<object type="image/svg+xml" data="invoice.svg">
  <img src="invoice.png" alt="Bitmap replacement">
</object>

Fallback notes: Always loads the bitmap image in modern browsers. Se the latest solution.

Fallback for inline SVG

The solution is the same as for SVG as <img>

<svg viewBox="0 0 100 100">
  <!-- SVG code -->
  <image src="fallback.png" xlink:href="" width="128" height="128">
</svg>

But there is one more trick with SVG <description>, you can put in it any valid HTML including textual fallback or an image.

<svg viewBox="0 0 100 100">
  <!-- SVG object code -->
  <desc><img src="discount.png" width="128" height="128" /></desc>
</svg>

Fallback notes: And again the same issue as with an <object> fallback - it loads fallback image in modern browsers.

Fallback for SVG as CSS background-image

The trick here is to find CSS syntax that is supported by browsers that support SVG, but not by the older browsers. linear-gradient and multiple backgrounds do the trick:

.svg-icon-bg-fallback {
  background: url(wallet.png);
  background: url(wallet.svg), linear-gradient(transparent, transparent);
}

Fallback notes: Slight performance hit on modern browsers from having to calculate the transparent gradient.

Bulletproof CSS only fallback

All the details about this fallback can be found in awesome CSS-Tricks article.

<!DOCTYPE html>
<html>
<head>
    <title></title>
    <style>
     /* Set initial style for older browsers by default */
     .svg-icon-that-works {
         background: url('computer.png');
         width: 128px;
         height: 128px;
     }

     /* This one will be overwriten in modern browsers */
     .fallback {
     }
    </style>
</head>
<body>

  <svg xmlns="http://www.w3.org/2000/svg" width="0" height="0">
    <style>
    <![CDATA[
      /* Modern browsers will apply this style when older will ignore it */
      .fallback { background: none; background-image: none; display: none; }
    ]]>
    </style>
  </svg>

  <!-- inline svg -->
  <svg xmlns="http://www.w3.org/2000/svg" width="40" height="40">
  <switch>
     <g><!-- SVG object code --></g>
     <foreignObject>
         <span class="svg-icon-that-works fallback"></span>
     </foreignObject>
  </switch>
  </svg>

  <!-- external svg -->
  <object type="image/svg+xml" data="computer.svg">
    <div class="svg-icon-that-works fallback"></div>
  </object>

</body>
</html>
